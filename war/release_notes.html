Release Notes

Version 0.1, 29 Jan 2014

First release. I wrote a blog post about this at http://jdesbonnet.blogspot.ie/2014/02/a-fine-stretch-in-evenings.html. The source code is available at GitHub: https://github.com/jdesbonnet/a-fine-stretch-in-the-evenings

Known problems:

  • uses timezonedb.com web API to deduce local timezone form latitude,longitude. This may take a few seconds and may on fail on occasion. I'm looking for a good alternative that doesn't involve an external web API.
  • If the local timezone cannot be deduced, will default to UTC which will cause chart rendering problems for zones far from the 0 meridian.
  • Because of the two items above, rendering charts near the poles is problematic.
