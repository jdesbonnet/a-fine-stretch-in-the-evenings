Version 0.1, 29 Jan 2014

First release. I wrote a blog post about this at http://jdesbonnet.blogspot.ie/2014/02/a-fine-stretch-in-evenings.html.

It makes extensive use of SVG (part the the HTML5 spec) so it certainly won't work with old browsers. It has been tested with the latest (at time of release) Google Chrome and Firefox. From experience it probably won't work with Internet Explorer (I don't have a copy handy to test right now).

The source code is available at GitHub: https://github.com/jdesbonnet/a-fine-stretch-in-the-evenings

Known problems:

  • uses timezonedb.com web API to deduce local timezone from latitude,longitude. This may take a few seconds and may on fail on occasion. I'm looking for a good alternative that doesn't involve an external web API.
  • If the local timezone cannot be deduced, will default to UTC which will cause chart rendering problems for zones far from the 0 meridian.
  • Because of the two items above, rendering charts near the poles is problematic.
  • Map allows markers to be dragged off-map which causes weird charts for non-existent latitudes.
  • If downloading source, I deliberately left out _key.jsp which holds the timeszonedb.com API key. That file should contain the following: <%! String timezoneDbKey="replace-with-key-here"; %>
